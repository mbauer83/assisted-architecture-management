## Resume protocol

1. Read PLAN §3 (locked decisions) first — D4 (no behaviour change) and D5 (contract test)
   are load-bearing.
2. Find the first unticked WU whose dependencies are ticked.
3. Gates before every commit (one at a time, never concurrent): `uv run python -m pytest -q
   -n auto`; `uv run ruff check src/ tests/`; `uv run zuban check`. No frontend change is
   expected — the GUI already has its Effect schemas; if a `response_model` forces a GUI
   schema change, that is a found contract drift (D4) — record it here and fix the shape.
4. Backend changes are inert until the owner restarts `arch-backend`; `/openapi.json` is
   regenerated by FastAPI at boot, so the live-spec check is restart-gated — queue it.
