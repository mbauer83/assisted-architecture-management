## Markdown File-Based Approach

The strategic decision to use markdown files in a directory structure as the primary storage format for architectural knowledge, rather than a database-only approach. This choice prioritizes: human readability (markdown renders everywhere), LLM accessibility (text-based, efficiently tokenized), git trackability (text diffs, per-file history), and transparency (artifacts visible in any file explorer). A runtime SQLite index provides the querying and graph traversal performance that flat files alone cannot deliver, while the files remain the authoritative source of truth.
