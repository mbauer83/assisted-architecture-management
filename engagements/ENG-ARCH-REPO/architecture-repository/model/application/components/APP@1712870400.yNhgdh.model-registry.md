## Model Registry

In-memory index of entity and connection metadata, built by scanning the filesystem. Provides fast lookup of entities by ID, type, domain, and keyword; connections by source, target, and type. Used by the verification system for reference resolution and by the write system for duplicate detection. The registry is rebuilt on startup by scanning the `model/` directory tree and parsing `.outgoing.md` files.
