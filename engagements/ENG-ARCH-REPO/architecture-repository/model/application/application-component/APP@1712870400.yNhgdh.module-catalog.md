## Module Catalog

Immutable catalog of loaded ontology modules, built once at startup from YAML definitions by ModuleCatalogBuilder. Provides fast lookup of entity types, connection types, diagram types, and their metadata. Injected into RuntimeCatalogs and distributed to all application consumers via dependency injection from the composition root. Runtime consumers use the focused catalogs (OntologyCatalog, ConnectionSemantics, DiagramTypeCatalog) — narrow views backed by the Module Catalog. Previously called "Model Registry" (the ModuleRegistry predecessor); renamed to ModuleCatalog in Phase B of the hexagonal conformance refactor.
