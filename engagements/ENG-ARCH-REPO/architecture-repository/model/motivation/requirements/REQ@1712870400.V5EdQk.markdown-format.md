## Markdown Format

All entity, connection, and document files shall use markdown format with YAML frontmatter. Markdown is simultaneously human-readable (renders in any editor, IDE, or browser), LLM-friendly (efficiently tokenized, widely represented in training data), and git-friendly (text-based diffs, merge-capable). Section delimiters (`<!-- §content -->`, `<!-- §display -->`) partition files into machine-parseable regions without breaking markdown rendering.
