## Configurable Frontmatter Schemata

The system shall support configurable frontmatter schemata specific to each of the three file types: entity files, connection files (.outgoing.md), and diagram files. Each schema defines which frontmatter fields are required, optional, and valid. Fields required by core tooling and processes are mandatory and cannot be removed or overridden — the schema can only extend (add custom fields). Schema definitions shall be stored as git-tracked configuration files, allowing different engagements or the enterprise repository to add custom frontmatter fields (e.g., domain-specific classifications, compliance markers) while maintaining a validated, machine-parseable structure. The verification tools shall validate artifacts against the applicable schema.
