## Git-Based Repository Configuration

The system shall store all repository configuration in git-tracked files, supporting a two-level configuration hierarchy: enterprise-level configuration (defaults, constraints, shared settings) and engagement-level configuration (local overrides, project-specific extensions). Configuration shall cover: frontmatter schema definitions, valid entity types and connection types, directory structure conventions, verification rule sets, and tool behavior settings. Git-based configuration ensures that configuration changes are versioned, reviewable, and auditable alongside the architecture content they govern.
