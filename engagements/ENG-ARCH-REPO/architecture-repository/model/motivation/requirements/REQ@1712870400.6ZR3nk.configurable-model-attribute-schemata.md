## Configurable Model Attribute Schemata

The system shall support configurable attribute schemata for file content sections, defined per entity-type and per connection-type. Attribute schemata govern the valid attributes in Properties tables (for entities) and in connection metadata blocks (for connections in .outgoing.md files). The default is a free schema — no required attributes beyond the core frontmatter. Custom schemata are defined in git-tracked configuration files and can specify required attributes, optional attributes, allowed values, and validation rules per entity-type or connection-type. The verification tools shall validate content section attributes against the applicable attribute schema.

## Properties

| Attribute | Value |
|---|---|
| Priority | Should |
| Category | Configurability |
| Distinction | Separate from frontmatter schemata — frontmatter is per file-type (entity/connection-file/diagram), attribute schemata are per entity-type and connection-type |
