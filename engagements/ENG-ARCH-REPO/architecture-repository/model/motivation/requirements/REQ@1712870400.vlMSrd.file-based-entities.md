## File-Based Entities

Each architectural entity shall be represented as a single markdown file in a directory structure organized by ArchiMate NEXT domain and element type. The filesystem layout (`model/{domain}/{type-plural}/`) serves as the primary storage and provides immediate navigability for both humans (via file explorer) and AI agents (via directory listing). This file-per-entity approach ensures git-trackability, enables fine-grained version history, and avoids the opacity of database-only storage.
