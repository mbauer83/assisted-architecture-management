## List-view for entities

The GUI frontend must provide a list-view for entities. Relationships of specialization, aggregation, and composition must have the aggregated / specializing / composing rows duplicated under each parent and indented to show a tree-like view. This list must be filterable by archimate-domain and entity-type, as well as sortable by type and by number of connections (total, incoming, symmetric, outgoing).
