## Two-Tiered Repository

The system shall support a two-tiered repository structure: an enterprise repository containing globally reusable artifacts (cross-cutting guidelines, enterprise-wide models, reusable building blocks), and per-engagement/project repositories containing locally relevant work (draft artifacts, project-specific models, internal iteration artifacts). This separation allows teams to work autonomously within their engagement scope while maintaining a curated body of enterprise-wide architectural knowledge. Artifacts flow from engagement to enterprise via the promotion mechanism.
