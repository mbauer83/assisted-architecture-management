## File-Based Artifacts

Each architectural artifact shall be represented as a single markdown or puml file in a directory structure organized by types and sub-types The filesystem layout serves as the primary storage and provides immediate navigability for both humans (via file explorer) and AI agents (via directory listing). This file-per-entity approach ensures git-trackability, enables fine-grained version history, and avoids the opacity of database-only storage.
