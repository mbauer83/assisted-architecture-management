## Verified Referential Integrity for Model, Diagrams, and Documents

The system shall verify that all references between artifacts resolve correctly. Entity IDs referenced in .outgoing.md connection files must correspond to existing entities. Aliases used in PUML diagrams must match registered entities. Cross-references in documents must point to valid artifacts. Status conformity shall also be enforced: draft entities and connections may only appear in draft diagrams. Verification runs both incrementally (at write time) and as a full-model sweep.
