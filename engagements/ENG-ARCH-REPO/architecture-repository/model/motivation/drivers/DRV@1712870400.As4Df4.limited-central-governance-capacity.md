## Limited Central Governance Capacity

Central architecture governance teams cannot scale linearly with the number of development teams, AI agents, and concurrent engagements. Traditional architecture review boards become bottlenecks when code generation velocity outpaces review capacity. The architecture repository must enable governance to be partially automated (through verification, referential integrity checking, and status-conformity enforcement) and partially distributed (through tool-mediated authoring that enforces standards at write time rather than at review time).

## Notes

This driver motivates the write-access-only-via-tools requirement and the automated verification capabilities, which shift governance from human gatekeeping to tool-enforced standards.
