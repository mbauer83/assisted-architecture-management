## Increasing Need for Effective LLM Access

As LLM-based coding agents become primary contributors to codebases, architectural knowledge must be accessible in formats that LLMs can efficiently ingest and produce. Binary formats, proprietary tool databases, and diagram-only representations are opaque to language models. The architecture repository must use text-based, structured formats (markdown, YAML frontmatter, PlantUML) that allow AI agents to discover, understand, and author architectural content through tool interfaces (MCP, CLI) with the same fidelity as human architects working through a GUI.

## Notes

This driver is the reason the repository uses markdown with structured frontmatter rather than a database-only approach. It also motivates the MCP interface and the dual human/AI design philosophy that runs through the entire system.
