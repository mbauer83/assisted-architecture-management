## Developer

A software developer (human or AI agent) who consumes architectural models for development guidance — understanding component boundaries, interface contracts, technology choices, and design rationale. Developers query the repository to discover relevant entities, navigate cross-domain connections, and ensure their implementation aligns with architectural intent. They may also author artifacts within their scope via the tool interfaces.

## Properties

| Attribute | Value |
|---|---|
| Category | Individual |
| Concerns | Development guidance clarity, discoverability of relevant artifacts, alignment between code and architecture, technology stack decisions |

## Notes

Developers interact primarily through discovery and querying capabilities. AI-based developers (coding agents) access the repository through MCP tools to retrieve context for spec-driven development.
