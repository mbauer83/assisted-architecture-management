## Configuration Service

Manages repository configuration by reading and merging git-tracked configuration files at enterprise and engagement levels. Provides the resolved configuration (frontmatter schemata per file-type, attribute schemata per entity-type and connection-type, valid entity/connection types, directory conventions, verification rules) to all other services and components. Configuration resolution follows an inheritance model: enterprise config provides defaults, engagement config may override or extend. Attribute schemata default to a free schema (no required attributes) unless explicitly configured for a given type.
