## Validate Datatype–Data Object Relationship Consistency

Evaluates the bidirectional semantic correspondence rule (Plan §3.2) on every datatype-diagram write: for each dt-* edge whose both endpoints are bound to Data Objects, verifies that the edge is connection-bound to exactly one backing model connection with matching relationship_kind, compatible direction, and correct endpoints. Emits E330 (missing backing) and E331 (non-corresponding backing) with ontology-derived permitted kinds and quick-fix payload. Correspondence is derived from ontology metadata — no hardcoded dt-*→archimate-* mapping.
