## Create Diagram

The process of creating a new architectural diagram: generate a unique ID, populate frontmatter (including diagram-type), write the PlantUML source file with the appropriate !include for the macro library and stereotype definitions, validate PUML syntax via the PlantUML engine, verify that all referenced entity aliases exist in the model, and update the runtime index. The diagram file is placed in the diagram-catalog directory.

## Properties

| Attribute | Value |
|---|---|
| Trigger | Author request via authoring service |
| Output | .puml file in diagram-catalog + updated index |
