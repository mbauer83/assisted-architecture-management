## Index Repository

The process of extracting metadata from all repository artifacts and building (or updating) the SQLite runtime index. Scans entity files for frontmatter, parses .outgoing.md files for connection graph edges, extracts keywords, and optionally pulls git history for volatility metrics. Supports both full re-index and incremental update (based on file modification timestamps). The index is a disposable derived artifact — always regenerable from source.

| Attribute | Value |

| Trigger | Explicit request, post-write hook, or startup |
| Output | Updated SQLite index database |
