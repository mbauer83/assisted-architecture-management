## Verify Model

The process of running full verification across the entire architectural model: scan all entity files for frontmatter compliance, validate content-section attributes against per-type attribute schemata (when configured), parse all .outgoing.md files and verify connection validity, check all diagram files for syntax correctness and alias resolution, verify document cross-references, and enforce status conformity rules. Produces a structured verification report with errors, warnings, and summary statistics.

## Properties

| Attribute | Value |
|---|---|
| Trigger | Explicit request via validation service, or CI/CD pipeline |
| Output | Verification report (errors, warnings, pass/fail) |
