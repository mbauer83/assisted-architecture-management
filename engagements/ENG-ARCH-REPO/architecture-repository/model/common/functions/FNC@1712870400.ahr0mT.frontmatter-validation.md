## Frontmatter Validation

Validates YAML frontmatter against the applicable schema for the artifact type. Checks required fields are present, field values conform to allowed enums (status, artifact-type), artifact-id matches the filename stem, version follows semver, and any custom fields defined in the repository configuration are valid. Returns structured validation results with field-level error details.

## Properties

| Attribute | Value |
|---|---|
| Input | Parsed frontmatter dict + schema definition |
| Output | Validation result (pass/fail with field-level errors) |
