## Scope

All motivation-domain content in this model (stakeholders, drivers, assessments, goals,
outcomes, principles, requirements) and the application-component traceability convention
the conformance views evaluate.

## Motivation

Reviews kept re-arguing the same semantics per edge: whether assessment mediation counts
as a driver-to-goal trace, when associations may become influences, and what "traced"
means for a component. Recording the conventions once makes conformance mechanically
checkable and keeps future passes from double-counting legitimate mediation as gaps.

See requirement [Support Models, Diagrams, and Documents](../../../projects/motivation-narrative/model/motivation/requirement/REQ@1712870400.HR7AGz.support-models-diagrams-documents.md)

## Summary

Assessment mediation is an accepted driver-to-goal trace; direct influence edges are the
exception and must state their causal claim. Stakeholders link to both drivers and goals.
Associations are only retyped to influence when a direction is already stated. Component
conformance means a forward structural chain to a goal or requirement.

## Specification

### Driver-to-goal tracing

`driver —influence→ assessment —association→ goal` is the preferred, richer trace because
the assessment names the judged condition the goal responds to. A direct
`driver —influence→ goal` edge is additionally warranted only when (a) no
assessment-mediated path exists AND (b) the causal claim is real; such an edge must carry
a one-sentence description stating the claim.

Current dispositions (all five drivers have a retained trace path; no direct edges were
added):

* Increasing Demand for More Autonomy of Teams and Systems — direct influence to five goals.
* Rising Software Complexity and Interdependence — direct influence to two goals.
* AI-Assisted Development as Dominant Production Mode — assessment-mediated (e.g. via "AI
  Code Generation Volume Outpaces Architectural Planning and Review Capacity" into three
  goals).
* Safety, Security and GRC Capability Gap for Small Teams — assessment-mediated (via
  "Assurance Cannot Keep Pace with Agentic Development Velocity" and "Assurance
  Disconnected From Architecture Loses Traceability" into the assurance goals).
* Expanding Software and AI Assurance Obligations — assessment-mediated (via "Assurance
  Disconnected From Architecture Loses Traceability" into "Provide First-Class Assurance
  over the Architecture Model").

### Stakeholder legs

Every stakeholder should be associated with one or more drivers AND one or more goals.
The stakeholder–driver hub uses `archimate-association` for consistency; use influence
only where a directional claim is stated in the description.

### Retyping associations

An `archimate-association` may be retyped to `archimate-influence` ONLY where the
connection's description already states a direction. Retyping is remove-and-recreate with
the same endpoints and description (a connection's type cannot be edited in place); the
new description must preserve the original provenance. The three driver–driver
associations currently carry no descriptions, so none were retyped.

### Component-conformance convention (shared with the traceability views)

An application component conforms iff a FORWARD structural chain
(composition/aggregation/assignment/realization, direct or derived up to 4 modeled hops)
reaches a goal or requirement — the same semantics the component-traceability-gaps
viewpoint evaluates. A part does NOT inherit the whole's trace (composition is not
traversed against its direction), and serving chains do not count as realization.
