## Assurance Write Queue

Internal FIFO single-writer async queue that serializes all mutations to the confidential assurance store. Every write operation — node creation, edge addition, baseline sealing, analysis deletion — is submitted as a callable to this queue and executed by a dedicated drain worker. Parallel writes from concurrent MCP tool calls and REST handlers block at the queue entry point, never at the store level, eliminating SQLite write-ahead-log contention.
