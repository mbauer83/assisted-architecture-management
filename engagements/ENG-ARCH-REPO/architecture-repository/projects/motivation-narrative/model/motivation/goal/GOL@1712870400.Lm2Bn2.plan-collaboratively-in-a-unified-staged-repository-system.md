## Plan Collaboratively in a Unified, Staged Repository System

Architects and stakeholders plan together and add their output to a unified, staged system of architecture repositories — one enterprise repository plus multiple team/project/engagement repositories as needed — with explicit promotion between stages and cross-stakeholder validation on a shared model. This staged structure is what lets many teams and engagements work at agentic velocity while remaining unified in effort — sharing plans, governance and validated architectural knowledge through the enterprise repository rather than diverging into isolated, unreconciled contexts.
