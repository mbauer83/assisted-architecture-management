## Architecture Work Accessible Through Both Human and Agent Interfaces

All core architecture discovery, authoring, and verification work is reachable through both human interfaces (GUI) and agent interfaces (MCP, CLI, REST, RAG), ensuring the architecture repository serves both human and automated consumers without claiming full capability parity across every feature.

## Properties

| Attribute | Value |
|---|---|
| Priority | Must |
| Rationale | The architecture repository must serve both human stakeholders working via GUI and automated agents working via MCP, CLI, or REST — single-interface design would exclude one class of consumer and undermine the system's mission of enabling collaborative and agentic architectural work |
