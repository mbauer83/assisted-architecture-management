## Concurrent Reads, Serialized Writes

In any deployment (local or team-serving) all store reads execute concurrently across threads via a per-thread WAL connection pool, while all store writes are strictly serialized through a single-writer queue. This eliminates file-write races and SQLite contention without blocking readers. The constraint applies uniformly to both the architecture model store (Write Request Queue + SQLite index) and the confidential assurance store (Assurance Write Queue + Assurance Read Connection Pool).
