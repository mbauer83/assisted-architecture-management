## Repository Format Upgrade

The system supports version-aware, idempotent migration of a deployment's persisted formats across software versions (arch-repair upgrade) — covering repositories AND deployment-level operational surfaces: the guidance cache, the signal stores, and the viewpoint declaration grammar. Dry-run reporting by default, a hard backend-not-serving gate before any write, auto-migratable findings applied atomically, non-migratable findings reported with manual instructions — never silently skipped. A previous-release deployment upgrades safely and identically whether run locally or from the container entrypoint.
