## Ontology-Driven Assurance Edge Authoring

Legal assurance edge types are derived from the assurance ontology module's exhaustive permitted-relationship matrix — the single source — and enforced server-side on every edge create: an illegal (source type, target type, connection type) triple is rejected with a typed error carrying the pair's full legal set, and no authoring surface carries its own literal type list.
