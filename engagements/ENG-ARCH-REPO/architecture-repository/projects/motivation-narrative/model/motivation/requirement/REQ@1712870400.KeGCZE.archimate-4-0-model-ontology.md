## ArchiMate 4.0 Model Ontology

The system shall adopt the ArchiMate 4.0 ontology as its architectural modeling framework. Key differences from ArchiMate 3.x: "layers" are renamed to "domains" (reflecting that they are not strict abstraction layers); all behavioral elements (services, processes, functions, events, collaborations) and roles are placed in a shared "common" domain rather than being duplicated across business/application/technology. The system shall enforce valid connection types between element types within and across domains following ArchiMate 4.0 relationship rules.
