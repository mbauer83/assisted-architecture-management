## Repository Authoring Policy: Required Attribute Defaults

Per-repo `.arch-repo/config.yaml` declares `required_defaults_policy` (`strict` or `non-strict`). Under `strict`, every required attribute in an attribute schema must declare a valid default; an invalid declared default is always a hard startup error regardless of policy. The policy is validated at backend startup across all configured repos.

## Notes

Implemented in `src/application/_startup_schema_policy.py` (SchemaPolicyError, validate_schema_policy); wired into backend startup via `_run_startup_validations`. New repos scaffold `non-strict` via the engagement repo template. The three project repos are set to `strict`.
