## Viewpoint-Based Model Presentation

The system supports executable, on-demand viewpoints: a declarative scope/selection over the model, rendered in one of several representations (graph exploration, catalog table, matrix, ad-hoc diagram) with capability-checked styling, without persisting new artifacts. This includes impact analysis — selecting a population by direct-or-derived relationship traversal, so a viewpoint can answer "what is affected if this element changes?" (or "what does it depend on?") by composing indirect relationships across intermediate elements at query time, again without persisting anything.
