## Typed Attribute Persistence and Editing

Property values are stored as canonical lexical forms in model files and decoded using the attribute's schema type. Supported kinds: `string`, `integer`, `number`, `boolean`, string-backed `enum`, and homogeneous scalar `array`. Schema-driven decode applies throughout the write path (formatter, parser, verifier, REST, MCP). The GUI renders type-appropriate controls for schema-declared attributes (select for enum, checkbox for boolean, numeric input for integer/number, tag editor for array) and a type-selector for ad-hoc attributes. Required attributes block create/save until filled.
