## Query Binding Evaluator

Evaluates a viewpoint query's typed let-bindings, parameters, and item-scoped derived attributes so the primary selection can compare against a second, independently-selected population without a formula or text language. Bindings are evaluated once over the same entity/connection population machinery the primary query uses (no second scan path); result types are checked statically, so a mistyped reference is a save-time error rather than a runtime surprise. Parameters resolve typed inputs at execution time; derived attributes reduce incident connections/endpoints or relationship-derivation facts to one value per matched item.
