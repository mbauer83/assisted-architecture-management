## Repository Upgrade Framework

Hexagonal upgrade framework backing arch-repair upgrade. It orchestrates multi-target operational upgrades — repositories, the deployment guidance cache, and both signal stores (co-located SQLCipher and public SQLite) — through a step registry with per-target transactional units and resumable, ordered application: all-target discovery and preflight before any write, then ordered, per-target-atomic, idempotent migration. It is explicitly NOT a distributed transaction — a failure after an earlier target has committed yields an accurate partial report and a safe resuming rerun. Auto-migratable findings apply atomically against the repo's format-contract version; non-migratable ones are reported with manual instructions, never silently skipped.
