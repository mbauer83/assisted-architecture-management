## Query Engine

Supports full-text search, metadata-based querying, graph-based relationship traversal, and viewpoint execution over the SQLite runtime index. Translates high-level query requests (from MCP, CLI, or REST interfaces) into SQL queries against the index tables, and a declared Viewpoint Definition's scope/selection into one of the execution representations (exploration, table, matrix, diagram). Supports TF-IDF ranking for full-text results, structured filtering/sorting for metadata queries, and recursive CTE-based traversal for graph queries. For viewpoint execution it composes two internal modules: the Relationship Derivation Engine (indirect/derived relationships across intermediate elements) and the Query Binding Evaluator (typed let-bindings, parameters, and item-scoped derived attributes). Viewpoint execution is reference-integrity aware: it reports every authored reference that no longer resolves against the current model (retired types, deleted specialization slugs, unresolvable attribute paths, dangling entity-id anchors) and keeps results honest — a query with a broken reference suppresses absence claims, so an empty result can never read as a clean bill of health.
