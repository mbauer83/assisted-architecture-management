## Batch Transaction Manager

Application component that coordinates scalable staged write transactions by combining indexed reference-blocker lookups, tracked-path manifest derivation, overlay-backed verification, copy-on-write staging when filesystem materialization is required, and atomic M4 publication. Intentionally carries no direct requirement-realization connection: it is a write-pipeline internal composed into the Architecture Backend and governed by the backend's requirements.
