## Specialization Profile

The technical realization: `.arch-repo/specializations.yaml` declares each specialization (entity and connection kinds, two-tier repo declarations plus a module-shipped informative library) with its own inline `attributes:`, an optional dedicated `attributes.{type}.{slug}.schema.json` (or `connection-metadata.{type}.{slug}.schema.json`) attachment, and optional `profiles:` bindings by name. Named, reusable profiles live in `.arch-repo/profiles.yaml` (format version plus per-profile content version), shared across specializations; the shipped module may also ship a registry, and a repo-level profile of the same name overrides it. The effective schema merges base, bound profiles in declaration order, then the specialization's own contribution last.
