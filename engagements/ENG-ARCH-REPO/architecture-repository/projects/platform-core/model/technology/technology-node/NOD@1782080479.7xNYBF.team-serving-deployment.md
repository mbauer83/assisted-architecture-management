## Team-Serving Deployment

A network-accessible deployment of the Architecture Management Platform that serves a team of concurrent users — multiple architects, product owners, and AI agents — simultaneously via the GUI and MCP interfaces. In this mode the Architecture Backend runs as a long-lived process bound to a network address, and the assurance store must handle concurrent reads from multiple GUI sessions and agent threads while serializing all writes.
