## Attribute Schema Validation

Validates content-section attributes against the applicable attribute schema for the entity-type or connection-type. For entities, checks the Properties table attributes; for connections in .outgoing.md files, checks connection metadata blocks. Attribute schemata are defined per entity-type and per connection-type in git-tracked configuration files. The default is a free schema (no required attributes). When a schema is defined, this function verifies required attributes are present, optional attributes conform to allowed values, and no disallowed attributes appear. Returns structured validation results with attribute-level error details.

## Properties

| Attribute | Value |
|---|---|
| Distinction | Separate from frontmatter validation — operates on content sections, not YAML frontmatter |
| Input | Parsed content attributes + attribute schema definition |
| Output | Validation result (pass/fail with attribute-level errors) |
