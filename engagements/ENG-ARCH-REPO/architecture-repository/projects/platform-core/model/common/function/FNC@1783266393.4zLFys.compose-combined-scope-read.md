## Compose Combined-Scope Read

Stateless read-time composition over the two canonical per-repo artifact indexes that serves a combined engagement+enterprise read without holding any state of its own. Realizes combined-scope (repo_scope="both") reads by delegating to the two canonical per-repo Artifact Index instances and composing their results at read time — it is a read-time composition, not a cache, and holds no state of its own, so it cannot itself go stale. Search and listing methods (full-text search, list_entities and siblings) merge both sides' results (globally re-sorted, or per-kind-capped for search); single-artifact lookups (get_entity and siblings) fall back from engagement to enterprise, since a fallback only ever needs one side to actually resolve. This asymmetry is deliberate: merges are needed wherever both sides could each contribute results, fallbacks suffice wherever an id can only ever live on one side. A future maintainer adding cached/derived state directly to this composition (instead of to one of the two canonical instances) would reintroduce the staleness-bug class this design is immune to.
